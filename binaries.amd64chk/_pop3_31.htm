Create a Mailbox

For a user to receive e-mail, you must create a unique mailbox for that user. If you are using Active Directory integrated authentication or local Windows accounts authentication, you can also create an associated user for the mailbox.

To create a mailbox

  1. On the primary navigation bar, click E-mail.
  2. Click Domains and Mailboxes.
  3. In the Name column, select the check box next to the domain that contains the mailbox you want to create.
  4. Click Mailboxes.
  5. Click New.
  6. Click Create associated user for this mailbox if you are using Active Directory or local Windows accounts authentication. You must select this check box if you are using Active Directory integrated authentication or local Windows accounts authentication when creating a mailbox. If the check box is already selected, clear it only if an account already exists with the same name as the mailbox you want to create.
  7. In User Name, type the name of the mailbox to be created.
  8. In Password, type the password for the new mailbox.
  9. In Confirm Password, retype the password you entered in Password, and click OK.

Notes

  • To perform this procedure, you must be a member of the Administrators group on the local computer, or you must have been delegated the appropriate authority. If the computer is joined to a domain, members of the Domain Admins group might be able to perform this procedure.
  • The Create associated user for this mailbox option is only available if the server is using Active Directory integrated or local Windows accounts authentication.
  • If you are using Active Directory integrated authentication, you must log on to the Active Directory domain, not the local computer, to perform this procedure.
  • The maximum length for a mailbox name is 20 characters for local Windows accounts authentication, and 64 characters for encrypted password file authentication or Active Directory integrated authentication. The minimum length is 1 character.
  • If you are using Active Directory integrated authentication or local Windows accounts authentication, Password and Confirm Password are unavailable until you select Create associated user for this mailbox.
  • If you are using Active Directory integrated authentication and have multiple POP3 e-mail domains, when you create a mailbox, consider whether the mailbox you are creating has the same name as an existing mailbox in another POP3 e-mail domain.

    Each mailbox is associated with an Active Directory user account, which has both a user logon name and pre-Windows 2000 user logon name. The user logon name is the name of the Active Directory user account and the POP3 mailbox. Usually, the pre-Windows 2000 user logon name is the same as the user logon name.

    If, however, you try to create a mailbox and user account when there is an existing user account with the same pre-Windows 2000 user logon name, a naming conflict will occur. Active Directory does not support multiple accounts with the same pre-Windows 2000 user logon name.

    If there is a naming conflict, the mailbox name and e-mail address will not be affected, but the account's pre-Windows 2000 logon name will be modified to avoid any naming conflict with the existing account. If you are using Secure Password Authentication, and a naming conflict has occurred, the pre-Windows 2000 logon name must be used for e-mail client authentication.

    You can determine the modified logon name by logging on locally to the server, by connecting to the server using Terminal Services, or by logging on to another computer in the Active Directory domain and going to the Active Directory Users and Computers snap-in in Administrative Tools.

    Go to the Users folder, right-click the user account, and then click Properties. Click the Account tab. The modified account name appears in User logon name (pre-Windows 2000).

    You must note the pre-Windows 2000 logon name and provide it to the user. For more information about Secure Password Authentication, see Secure Password Authentication.
