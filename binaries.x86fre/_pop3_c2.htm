Server Properties

From this page, you can set the authentication method, server port, event logging level, and root mail directory for the server running the POP3 service. You can also require Secure Password Authentication (SPA) for all client connections.

Authentication Method - Specifies the three different authentication methods that the POP3 service uses to verify users who connect to the mail server.

Choose Local Windows accounts authentication if:

  • Your mail server is not a domain controller.

    • -and-
  • You want to create user accounts on the server on which the POP3 service is installed.

Choose Active Directory Integrated authentication if:

  • Your mail server is a domain controller or a member of an Active Directory domain.

Choose Encrypted Password File authentication if:

  • Your mail server is not using Active Directory.

    • -or-
  • You do not want to have user accounts for the POP3 service on the local computer.

Server Port - Specifies the port on which the POP3 service monitors for e-mail clients and responds to connection requests. The port is configured by default to the Internet standard of 110. If you change this port, you must inform all users that you are using a non-standard port because most e-mail clients set port 110 as the default port for connecting to a server running the POP3 service.

Logging Level - Specifies how much information is recorded by the POP3 service in the event logs. You can use event log information for auditing and troubleshooting. The logging levels are:

  • None - No events are logged.
  • Minimum - Only critical events are logged.
  • Medium - Critical and warning events are logged.
  • Maximum - Critical, warning, and informational events are logged.

Require Secure Password Authentication (SPA) for all client connections - Select this option to require all e-mail clients to transmit the user name and password information using secure authentication. Select this option only if you are using Active Directory integrated authentication or local Windows accounts authentication.

Root Mail Directory - Specifies the mail store, a directory on the local file system or a Universal Naming Convention (UNC) path that the POP3 service uses to store e-mail messages. To set the mail store, provide a path to a directory on the local file system or a UNC path.

Important

  • You cannot set the mail store to the root directory of the hard disk, for example C:\, or to a directory in which files are currently in use.
