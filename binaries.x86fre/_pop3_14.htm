Mail Store

Important

  • If you change this parameter, you must stop and then restart the POP3 service and the IIS Admin service. You can manage these services from Services on the Administrative Tools menu, which you can access by logging on locally to the server or by connecting to the server using Terminal Services.

The mail store is a directory or Universal Naming Convention (UNC) path in which the POP3 service stores all e-mail until users retrieve and download it to their client computer. The POP3 service creates a directory for each of its domains. For each user with a mailbox in that domain, the POP3 service creates a user directory under the directory for the domain.

E-mail that a user receives is stored as an individual file within the user's directory until the user retrieves it using a POP3 e-mail client. The following is an example of the complete path to an e-mail message stored for user named someone:
C:\inetpub\mailroot\mailbox\example.com\P3_someone.mbx\P3593893.eml

The directory permissions and file permissions for each directory in the mail store are identical. When you configure the mail store, the permissions are set so that only local or domain administrators and the local network service, which the POP3 service is configured to run with, are assigned permissions to the directories. No other user is assigned read/write permissions.

The mail store must have access to adequate hard disk space. To ensure that the mail store functions correctly, estimate the amount of disk space your mail store will use based on the number of users on the server, the volume of e-mail that they will receive, and the average size of the e-mail they will receive.

You can use disk quotas to protect the server from situations where the mail store's disk usage might increase unexpectedly. Disk quotas monitor and control disk space that is used on NTFS file system volumes.

To set the mail store

  1. On the primary navigation bar, click E-mail.
  2. Click Server Properties.
  3. In Root Mail Directory, type the path to the directory of the mail store on the local hard disk. For example, C:\mailstore. Then, click OK.

Important

  • Depending on your authentication method, certain characters are not allowed in mailbox names. Periods are allowed, but they cannot be either leading or trailing. The following example illustrates the correct use of a period:

    john.someone@example.com

Notes

  • To perform this procedure, you must be a member of the Administrators group on the local computer, or you must have been delegated the appropriate authority. If the computer is joined to a domain, members of the Domain Admins group might be able to perform this procedure.
  • You must configure the mail store to use either a directory on the local hard disk or a Universal Naming Convention (UNC) path. The POP3 service does not support other storage options, such as mapped drives.
  • If you need to transfer the mail store to a new directory, you must move the mail store directory to ensure the directory retains the correct ownership; copying the mail store will not work.
  • You cannot set the mail store to the root directory of the hard disk, such as C:\, or to a directory in which files are currently in use.
  • The default mail store is system_drive:\Inetpub\mailroot\Mailbox.
  • Locate the mail store on an NTFS partition.
  • If you change the mail store from its default location or from a previous location, and there is stored e-mail, you must manually move the folders in which the e-mail is stored to the new location.
  • Physical access to a server is a high security risk. To maintain a more secure environment, you must restrict physical access to the server where the mail store exists.
